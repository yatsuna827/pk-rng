# lcg32 CLI 要件定義書

## 概要

ポケモン第3世代の疑似乱数アルゴリズム（LCG: Linear Congruential Generator）の計算を行うCLIツール。
与えられたseed値に対して各種計算を行い結果を出力する。

## コマンド名

```
lcg32
```

## サブコマンド一覧

| サブコマンド | 説明 |
|-------------|------|
| `next`      | seed値を指定回数進める |
| `prev`      | seed値を指定回数戻す |
| `index`     | seed値が基準seedから何番目かを計算 |
| `list`      | seed値から連続したseedを列挙 |
| `rand`      | seed値から乱数値（上位16ビット）を計算 |
| `randf`     | seed値から0.0〜1.0の浮動小数点乱数を計算 |

## 共通仕様

### 入力形式（seed値）

- 16進数（hex）形式のみ対応
- 以下の形式をすべて受け付ける（柔軟な入力）:
  - `DEADBEEF`（プレフィックスなし大文字）
  - `deadbeef`（プレフィックスなし小文字）
  - `0xDEADBEEF`（0xプレフィックス付き大文字）
  - `0xdeadbeef`（0xプレフィックス付き小文字）
  - `0XDEADBEEF`（0Xプレフィックス付き）
- 8桁以下であること（32bit符号なし整数の範囲内）

### 出力形式（seed値）

- デフォルト: 0xプレフィックスなし、大文字、8桁ゼロ埋め
  - 例: `DEADBEEF`, `00000000`
- `--prefix` / `-p` オプション指定時: 0xプレフィックス付き
  - 例: `0xDEADBEEF`, `0x00000000`

### グローバルオプション

| オプション | 短縮形 | 説明 |
|-----------|--------|------|
| `--help` | `-h` | ヘルプを表示 |
| `--version` | `-V` | バージョンを表示 |

### hex出力サブコマンド共通オプション

`next`, `prev`, `list`で使用可能。

| オプション | 短縮形 | 説明 |
|-----------|--------|------|
| `--prefix` | `-p` | 出力に0xプレフィックスを付ける |

## サブコマンド詳細

> **注意**: 各サブコマンドの使用例に記載されている出力値は説明のためのサンプルであり、実際の計算結果とは異なる場合があります。実装時は必ずライブラリの計算結果に従ってください。

### next

seed値を指定回数進める。負の値を指定した場合は戻る（`prev`と同等）。

```
lcg32 next <SEED> <COUNT> [OPTIONS]
```

#### 位置引数

| 引数 | 説明 | 必須 |
|------|------|------|
| `SEED` | 進める対象のseed値（hex） | ✓ |
| `COUNT` | 進める回数（負の値で戻る） | ✓ |

#### 使用例

```bash
$ lcg32 next 00000000 3
E97E7B6A

$ lcg32 next E97E7B6A -3
00000000

$ lcg32 next 0xDEADBEEF 100 --prefix
0x12345678
```

### prev

seed値を指定回数戻す。

```
lcg32 prev <SEED> <COUNT> [OPTIONS]
```

#### 位置引数

| 引数 | 説明 | 必須 |
|------|------|------|
| `SEED` | 戻す対象のseed値（hex） | ✓ |
| `COUNT` | 戻す回数 | ✓ |

#### 使用例

```bash
$ lcg32 prev E97E7B6A 3
00000000

$ lcg32 prev DEADBEEF 50 -p
0xABCDEF12
```

### index

seed値が基準seedから何番目かを計算する。

```
lcg32 index <SEED> [OPTIONS]
```

#### 位置引数

| 引数 | 説明 | 必須 |
|------|------|------|
| `SEED` | 計算対象のseed値（hex） | ✓ |

#### オプション

| オプション | 短縮形 | 説明 | デフォルト |
|-----------|--------|------|-----------|
| `--from` | `-f` | 基準となるseed値（hex） | `00000000` |

#### 出力形式

- 10進数の整数値

#### 使用例

```bash
$ lcg32 index E97E7B6A
3

$ lcg32 index DEADBEEF --from 12345678
98765432
```

---

### list

seed値から連続したseedを列挙する。

```
lcg32 list <SEED> <COUNT> [OPTIONS]
```

#### 位置引数

| 引数 | 説明 | 必須 |
|------|------|------|
| `SEED` | 開始seed値（hex） | ✓ |
| `COUNT` | 出力する件数 | ✓ |

#### オプション

| オプション | 短縮形 | 説明 | デフォルト |
|-----------|--------|------|-----------|
| `--separator` | `-s` | 区切り文字 | 改行（`\n`） |

#### 使用例

```bash
$ lcg32 list 00000000 5
00000000
00006073
E97E7B6A
52713895
31B0DDE4

$ lcg32 list 00000000 5 --separator ","
00000000,00006073,E97E7B6A,52713895,31B0DDE4

$ lcg32 list 00000000 3 -p
0x00000000
0x00006073
0xE97E7B6A
```

### rand

seed値を1つ進めた値から乱数値（上位16ビット）を計算する。

```
lcg32 rand <SEED> [OPTIONS]
```

#### 位置引数

| 引数 | 説明 | 必須 |
|------|------|------|
| `SEED` | 計算対象のseed値（hex） | ✓ |

#### オプション

| オプション | 短縮形 | 説明 | デフォルト |
|-----------|--------|------|-----------|
| `--mod` | `-m` | 剰余を取る値 | なし（0〜65535を返す） |
| `--hex` | `-x` | 出力を16進数にする | なし（10進数） |

#### 出力形式

- デフォルト: 10進数の整数値（0〜65535、`--mod`指定時は0〜m-1）
- `--hex`指定時: 大文字hex 4桁ゼロ埋め（0000〜FFFF）

#### 実装（疑似コード）

```
let ref = Lcg32(seed).to_ref()
result = ref.get_rand()        // --mod なし
result = ref.get_rand(m=m)     // --mod あり
```

#### 使用例

```bash
$ lcg32 rand 00000000
24691

$ lcg32 rand 00000000 --mod 100
91

$ lcg32 rand 00000000 --hex
6073

$ lcg32 rand 00000000 -m 256 -x
0073
```

---

### randf

seed値を1つ進めた値から0.0〜1.0の浮動小数点乱数を計算する。

```
lcg32 randf <SEED>
```

#### 位置引数

| 引数 | 説明 | 必須 |
|------|------|------|
| `SEED` | 計算対象のseed値（hex） | ✓ |

#### 出力形式

- 浮動小数点数（0.0 ≤ x < 1.0）

#### 実装（疑似コード）

```
let ref = Lcg32(seed).to_ref()
result = ref.get_rand_f()
```

#### 使用例

```bash
$ lcg32 randf 00000000
0.376815795898438
```

## エラーハンドリング

### 入力エラー

| エラー条件 | エラーメッセージ例 |
|-----------|------------------|
| seed値が不正な形式 | `Invalid seed format: "xyz"` |
| seed値が範囲外（8桁超過） | `Seed value out of range: "123456789"` |
| 必須引数の不足 | `Missing required argument: <COUNT>` |
| 回数が負の値（`prev`, `list`） | `Count must be a positive integer` |
